## Platform-Specific Knowledge

### Tandy 4000

#### Overview
- 386DX-16MHz business computer released 1988 - Source: [qmulIyfxlS4]
- Positioned as "AT class" business machine, not home computer - Source: [qmulIyfxlS4]
- Retail price $2599 (base) to $4300 (with 40MB hard drive) - Source: [qmulIyfxlS4]
- Very rare machine - few videos or documentation exist - Source: [qmulIyfxlS4]
- Chips and Technologies chipset (commodity 386 chipset) - Source: [qmulIyfxlS4]

#### Hardware Specifications
- 386DX processor at 16MHz (very fast for 1988) - Source: [qmulIyfxlS4]
- 32-bit data path to RAM - Source: [qmulIyfxlS4]
- 8 SIMM slots for memory expansion - Source: [qmulIyfxlS4]
- 1MB RAM standard, expandable to 2MB with 256K SIMMs - Source: [qmulIyfxlS4]
- Up to 16MB with 1MB SIMMs - Source: [qmulIyfxlS4]
- 3.5" high-density floppy drive (1.44MB) - first Tandy with this - Source: [qmulIyfxlS4]
- 10 expansion slots: 8 16-bit ISA, 2 8-bit ISA - Source: [qmulIyfxlS4]
- 32-bit RAM expansion slot for add-in card - Source: [qmulIyfxlS4]
- Optional 80287 math coprocessor socket - Source: [qmulIyfxlS4]

#### Power Supply
- Proprietary power connector (not standard AT) - Source: [qmulIyfxlS4]
- Can be adapted using ATX-to-AT converters with wire modifications - Source: [qmulIyfxlS4]
- Power supply wires: red=5V, purple=12V, white=-12V, white stripe=-5V - Source: [qmulIyfxlS4]
- Requires power good signal (5V) to boot - Source: [qmulIyfxlS4]

#### BIOS
- Phoenix Technologies BIOS version 1.3.0 (1987) - Source: [qmulIyfxlS4]
- Two 27C128 EPROMs (odd/even for 16-bit data bus) - Source: [qmulIyfxlS4]
- "Licensed to Tandy Corp" string in ROM - Source: [qmulIyfxlS4]
- ROM dump available on archive.org - Source: [qmulIyfxlS4]
- Chips and Technologies chipset compatible with other 386 boards - Source: [qmulIyfxlS4]

#### Operating System Support
- MS-DOS 3.2 (Tandy customized version) - NOT included in price - Source: [qmulIyfxlS4]
- OS/2 support (heavily marketed at time) - Source: [qmulIyfxlS4]
- Xenix (Unix System V) support - $600 extra - Source: [qmulIyfxlS4]
- Could function as multi-user system with Xenix - Source: [qmulIyfxlS4]

#### Motherboard Notes
- Date codes mostly 1987 despite 1988 release - Source: [qmulIyfxlS4]
- Four PAL chips for custom logic (may be irreplaceable) - Source: [qmulIyfxlS4]
- Integrated floppy controller (only integrated peripheral) - Source: [qmulIyfxlS4]
- Video card not included - extra cost ($130-300) - Source: [qmulIyfxlS4]
- Keyboard inhibit switch for security - Source: [qmulIyfxlS4]
- Mostly socketed ICs - Source: [qmulIyfxlS4]

#### Common Issues
- 16V tantalum capacitors on 12V rail fail short - Source: [qmulIyfxlS4]
- Corrosion in sockets prevents proper contact - Source: [qmulIyfxlS4]
- PAL chips are custom programmed - if damaged, board may be unrepairable - Source: [qmulIyfxlS4]
- Service manual with schematics available - Source: [qmulIyfxlS4]

#### Documentation
- Hardware reference manual: ftp://mindcandydvd.com/pub/tvdog/other_tandy_x86/4000trm.pdf - Source: [qmulIyfxlS4]
- 1988 Tandy catalog at colorcomputerarchive.com - Source: [qmulIyfxlS4]

### TRS-80 Model 3

#### Diagnostic ROM
- GitHub: github.com/misterblack1/trs80-diagnosticrom - Source: [4fuuyLiSgsE]
- Uses ROM A socket (primary bootstrap ROM) - Source: [4fuuyLiSgsE]
- 2364 mask ROMs not pin-compatible with EPROMs - need adapter PCB - Source: [4fuuyLiSgsE]
- Adapter works with 2364, 2332, 2716 sockets → 27128 or 2764 EPROMs - Source: [4fuuyLiSgsE]
- ROM B and ROM C not needed for diagnostic to run - Source: [4fuuyLiSgsE]

#### Diagnostic ROM Features
- Initial beep from cassette port confirms code execution - Source: [4fuuyLiSgsE]
- No beep = dead CPU, bad ribbon cable, or cassette port problem - Source: [4fuuyLiSgsE]
- Tests video RAM first (only 1KB), uses it as stack pointer - Source: [4fuuyLiSgsE]
- Can boot and display text with NO working DRAM - Source: [4fuuyLiSgsE]
- Uses March test: thorough stuck bit and cross-affect detection - Source: [4fuuyLiSgsE]
- Also uses DWF random number test (by Frank IZ8DWF) - Source: [4fuuyLiSgsE]
- March test detects page errors that simpler tests miss - Source: [4fuuyLiSgsE]
- Rising tone = test passed, falling tone = test failed - Source: [4fuuyLiSgsE]
- Display shows bit pattern (0-7) indicating which chip failed - Source: [4fuuyLiSgsE]
- VRAM failure: beeping pattern before text can display - Source: [4fuuyLiSgsE]

#### March RAM Test Algorithm
- Fill all memory with zeros, read each location, invert, write back - Source: [4fuuyLiSgsE]
- Run twice, should end with all zeros - Source: [4fuuyLiSgsE]
- Repeat with all ones, should end with all ones - Source: [4fuuyLiSgsE]
- Detects stuck bits and cross-talk between addresses - Source: [4fuuyLiSgsE]

#### Page Error Detection
- Simple bit pattern tests miss page errors - Source: [4fuuyLiSgsE]
- Page error: writing one address affects different page (256-byte boundary) - Source: [4fuuyLiSgsE]
- Causes unpredictable crashes, stack corruption - Source: [4fuuyLiSgsE]
- March test and random tests catch page errors - Source: [4fuuyLiSgsE]

#### Z80 vs 6502 Stack
- Z80: Stack pointer can be set to any memory location - Source: [4fuuyLiSgsE]
- 6502: Stack always at $100, needs working DRAM there - Source: [4fuuyLiSgsE]
- TRS-80 diagnostic uses VRAM as stack, avoids DRAM dependency - Source: [4fuuyLiSgsE]

#### Hardware Notes
- Maximum RAM capacity: 48K - Source: [_0XGFz67DIc]
- Uses Alps SKTC keyboard switches - Source: [_0XGFz67DIc]
- Monochrome CRT display - Source: [_0XGFz67DIc]
- Case has familiar silver paint color on bottom - Source: [_0XGFz67DIc]
- Cassette-only models lack internal power supply and drive cage structure - Source: [1EOJKuuxdQ0]
- 26-1062 SKU = 16K RAM with Level 2 BASIC, no floppy drives - Source: [1EOJKuuxdQ0]
- Single drive models: 26-1063, 26-1065, 26-1066 (different RAM configs) - Source: [1EOJKuuxdQ0]
- Level 1 BASIC with 4K RAM: SKU 26-1061 - Source: [1EOJKuuxdQ0]

#### Keyboard
- Alps SKTC switches: When exposed to dirt, debris can get inside and affect contacts - Source: [_0XGFz67DIc]
- Alps SKTC switches are plastic heat-staked together, making disassembly and contact plate cleaning difficult - Source: [_0XGFz67DIc]
- Keyboard layout differs slightly from Model 4 - Source: [_0XGFz67DIc]
- Alps SKCC keyboard (not SKTC) is most common on Model 3, very nice feeling - Source: [1EOJKuuxdQ0]
- Some Model 3 and Model 4 units have less desirable keyboard types - Source: [1EOJKuuxdQ0]

#### Power Supply
- Contains RIFA X2 capacitors that fail and release smoke - Source: [1EOJKuuxdQ0]
- Small capacitor near hot resistor and diode typically fails - replace preemptively - Source: [1EOJKuuxdQ0]
- Replace RIFA caps before powering on old units - Source: [1EOJKuuxdQ0]

#### Display Adjustments
- Brightness and contrast pots can become scratchy/gummy with age - Source: [1EOJKuuxdQ0]
- Clean pots with QD Electric Cleaner, then re-lubricate with DeoxIT Fader Lube - Source: [1EOJKuuxdQ0]
- Rear pot = contrast, front pot = brightness (black level) - Source: [1EOJKuuxdQ0]

#### Disk Drive Notes
- Original drives were single-sided - Source: [1EOJKuuxdQ0]
- Upgrade to double-sided drives was common - Source: [1EOJKuuxdQ0]
- TEAC drives commonly used - Source: [1EOJKuuxdQ0]
- Cassette-only models require cutting blank plates to install drives - Source: [1EOJKuuxdQ0]

#### Joystick Mod
- No native joystick support - Source: [1EOJKuuxdQ0]
- Common mod: Wire Atari-style joystick connector to arrow keys and space bar - Source: [1EOJKuuxdQ0]
- Joystick connector often mounted on side of case - Source: [1EOJKuuxdQ0]
- Enables games like "13 Ghosts" to work with joystick - Source: [1EOJKuuxdQ0]

#### Software
- 13 Ghosts: Character-based graphics game, works on Model 1 and Model 3 - Source: [1EOJKuuxdQ0]
- Sound output available via cassette port (add amplifier for internal speaker) - Source: [1EOJKuuxdQ0]

### TRS-80 Model 1

#### Overview and History
- Part of the "Trinity" first three personal computers with Apple II and Commodore PET (1977) - Source: [zHe7XwFozk0]
- Base RAM: 4K or 16K (4116 DRAM chips) - Source: [zHe7XwFozk0]
- Expansion interface adds 32K for total 48K - Source: [zHe7XwFozk0]
- Uses same 5-pin DIN connector for cassette, video, AND power (user unfriendly) - Source: [zHe7XwFozk0]

#### Motherboard Layout
- All ICs on underside of board - Source: [zHe7XwFozk0]
- Z80 processor - Source: [zHe7XwFozk0]
- Power input is AC, converted internally to voltage rails - Source: [zHe7XwFozk0]
- 5V regulator with heatsink - Source: [zHe7XwFozk0]
- -5V for RAM and +12V also generated - Source: [zHe7XwFozk0]
- RadioShack part numbers on chips make identification harder - Source: [zHe7XwFozk0]

#### Level II Basic Expansion
- Adds extra ROM socket board via ribbon cable - Source: [zHe7XwFozk0]
- Plugs into original ROM socket - Source: [zHe7XwFozk0]
- Uses 2332 ROMs (4K each) - Source: [zHe7XwFozk0]
- Three ROMs: total 12K (vs original 8K) - Source: [zHe7XwFozk0]
- Extra wires to motherboard for address bus and chip select - Source: [zHe7XwFozk0]
- Official Radio Shack mod dated 1978 - Source: [zHe7XwFozk0]
- Ribbon cable connection is flaky/fragile - Source: [zHe7XwFozk0]

#### Keyboard
- Original Model 1 did not have numeric keypad - Source: [zHe7XwFozk0]
- Keypad added later, just wires to same key switches - Source: [zHe7XwFozk0]
- Pressing "5" on keypad = same as "5" on main keyboard - Source: [zHe7XwFozk0]
- Rubber spacers hold keyboard in place, get cracked with age - Source: [zHe7XwFozk0]
- Spacers should be supple/squishy, not hard/cracked - Source: [zHe7XwFozk0]
- Can 3D print replacement spacers - Source: [zHe7XwFozk0]
- Keyboard connected via flimsy ribbon cable that corrodes - Source: [zHe7XwFozk0]
- Fix: Replace ribbon with IDE cable on pin headers - Source: [zHe7XwFozk0]

#### Video Memory and Lowercase Mod
- Standard: 7 chips of video RAM (7 bits) - Source: [zHe7XwFozk0]
- Video RAM is static RAM (2102 type chips) - Source: [zHe7XwFozk0]
- Missing 8th chip = no lowercase support - Source: [zHe7XwFozk0]
- Lowercase mod: Piggyback 8th 2102 chip onto existing one - Source: [zHe7XwFozk0]
- Bend two pins out, solder rest to existing chip - Source: [zHe7XwFozk0]
- Run wires to motherboard points - Source: [zHe7XwFozk0]
- Cut a trace for chip select - Source: [zHe7XwFozk0]
- Also requires keyboard mod for shift key support - Source: [zHe7XwFozk0]

#### Video Output
- 5-pin DIN connector - Source: [zHe7XwFozk0]
- Shield = ground - Source: [zHe7XwFozk0]
- One pin = video signal - Source: [zHe7XwFozk0]
- One pin = 5V supply for TV monitor - Source: [zHe7XwFozk0]
- 5V powers converter board inside monitor - Source: [zHe7XwFozk0]
- Video signal is very non-standard (not composite compatible) - Source: [zHe7XwFozk0]
- ~1.5V peak-to-peak - Source: [zHe7XwFozk0]
- Requires matching TRS-80 monitor or modification - Source: [zHe7XwFozk0]

#### Diagnostic ROM
- Works on Model 1 if 8 bits of video RAM present - Source: [zHe7XwFozk0]
- Uses video RAM as processor stack - Source: [zHe7XwFozk0]
- Initial beep from cassette port confirms code execution - Source: [zHe7XwFozk0]
- Can work with only 7-bit video RAM with modified ROM - Source: [zHe7XwFozk0]
- ROM A socket used (2364 or adapter needed) - Source: [zHe7XwFozk0]
- May need socket stacked to clear expansion connector - Source: [zHe7XwFozk0]

#### Expansion Interface
- Connects to rear expansion port - Source: [zHe7XwFozk0]
- Adds 32K RAM (total 48K) - Source: [zHe7XwFozk0]
- Adds disk drive interface - Source: [zHe7XwFozk0]
- Adds printer interface - Source: [zHe7XwFozk0]
- Adds serial interface - Source: [zHe7XwFozk0]
- Houses power supplies for both computer and itself - Source: [zHe7XwFozk0]
- Uses same AC power brick as computer - Source: [zHe7XwFozk0]

#### Common Failure: Video Output Circuit
- Video output may fail suddenly - Source: [zHe7XwFozk0]
- Symptom: Only vertical sync pulse remains, no horizontal - Source: [zHe7XwFozk0]
- CPU may still be running (address/data bus activity visible) - Source: [zHe7XwFozk0]
- Diagnostic ROM beep confirms CPU working even with no video - Source: [zHe7XwFozk0]
- Video circuit is independent of CPU operation - Source: [zHe7XwFozk0]

#### Power Supply Issues
- Original power supplies are noisy transformer-based designs - Source: [eyV7kaEMcfs]
- Buzzy and obtrusive sound from original PSU - Source: [eyV7kaEMcfs]
- Two separate power bricks: one for computer, one for expansion interface - Source: [eyV7kaEMcfs]
- Original design stores both power bricks inside expansion interface - Source: [eyV7kaEMcfs]
- Results in messy cables and two mains cords coming out - Source: [eyV7kaEMcfs]

- Single mains cable instead of two - Source: [eyV7kaEMcfs]

#### Overview

- Bank switch register controls which RAM page is active - Source: [azsTZ-Cmijs]

#### Diagnostic ROM (Ported Version)

- May work on TRS-80 Model 16, 16B, and Tandy 6000 in Z80 mode - Source: [azsTZ-Cmijs]

#### Diagnostic ROM Beep Codes
- No speaker in computer - uses floppy drive LED flashing - Source: [azsTZ-Cmijs]
- LED flashes 3 times during memory test - Source: [azsTZ-Cmijs]
- Head load solenoid clicks accompany LED flashes - Source: [azsTZ-Cmijs]
- If video works but LED doesn't flash: floppy controller issue - Source: [azsTZ-Cmijs]
- If LED flashes but no video: CPU working, display problem - Source: [azsTZ-Cmijs]

#### Stock ROM Diagnostics

- Uses primitive test algorithm - Source: [azsTZ-Cmijs]
- Tests floppy controller and some other components - Source: [azsTZ-Cmijs]
- Boot error "DC" = Disk Controller error - Source: [azsTZ-Cmijs]

#### ROM Socket
- Uses 2KB ROM (equivalent to 2716 EPROM) - Source: [azsTZ-Cmijs]
- Standard EPROMs are pin-compatible - Source: [azsTZ-Cmijs]
- Adapter board can use 2764 EPROM in place of 2716 - Source: [azsTZ-Cmijs]
- Some address lines may be tied high - load ROM at offset (1000h or 2000h) - Source: [azsTZ-Cmijs]

#### Maintenance Notes
- Very heavy machine - difficult to move - Source: [azsTZ-Cmijs]
- DeoxIT on slot connectors recommended - cards can be very stiff - Source: [azsTZ-Cmijs]
- Cards can be stuck in slots if not maintained - Source: [azsTZ-Cmijs]
