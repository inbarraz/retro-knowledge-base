## Platform-Specific Knowledge

### Voodoo 2 Series

#### Overview
- 3D-only accelerator (requires separate 2D card) - Source: [m3ETdhOuZ2w]
- Uses VGA pass-through cable design - Source: [m3ETdhOuZ2w]
- 2D card output connects to Voodoo 2 input - Source: [m3ETdhOuZ2w]
- Voodoo 2 output goes to monitor - Source: [m3ETdhOuZ2w]
- Card switches between 2D passthrough and 3D output automatically - Source: [m3ETdhOuZ2w]

#### Creative Labs CT6670
- Creative Labs branded Voodoo 2 card - Source: [m3ETdhOuZ2w]
- Uses 3dfx reference design with Creative branding - Source: [m3ETdhOuZ2w]
- Supports SLI (two cards for improved performance) - Source: [m3ETdhOuZ2w]
- 12MB RAM variant most common - Source: [m3ETdhOuZ2w]

#### Glide API and DOS Gaming
- 3dfx games use Glide API (not DirectX or OpenGL) - Source: [m3ETdhOuZ2w]
- DOS games need specific Glide .OVL overlay files - Source: [m3ETdhOuZ2w]
- Different games may need different Glide versions - Source: [m3ETdhOuZ2w]
- OVL files go in game directory - Source: [m3ETdhOuZ2w]
- May need to experiment with different Glide versions for compatibility - Source: [m3ETdhOuZ2w]

#### Voodoo 2 vs Voodoo Banshee
- Banshee combines 2D and 3D on single card - Source: [m3ETdhOuZ2w]
- Banshee has fewer texture units than Voodoo 2 - Source: [m3ETdhOuZ2w]
- Banshee can be fussy with some DOS games - Source: [m3ETdhOuZ2w]
- Voodoo 2 generally more compatible with DOS Glide games - Source: [m3ETdhOuZ2w]
- Voodoo 2 requires more cables and complexity - Source: [m3ETdhOuZ2w]

#### Common Issues
- Pass-through cable introduces slight signal degradation - Source: [m3ETdhOuZ2w]
- Some games crash or hang with wrong Glide version - Source: [m3ETdhOuZ2w]
- DOS mode more finicky than Windows 95/98 - Source: [m3ETdhOuZ2w]
- Memory issues in DOS with high conventional memory usage - Source: [m3ETdhOuZ2w]
